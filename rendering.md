# 描画のまとめ

HDRPやUE5と違って、URPにはボリュームテクスチャの描画機能がないため、自分で作る必要があります。
まあ、想像するより簡単なものですけどね。基本的にボリュームをスライスして、重ねて描画していきます。

この辺の詳細は好みのスタイルによるかと思いますが、私がやった方法を記載しますね。
結果的にハイブリッドなアプローチを採用しました。煙は通常の透明なボリュームとして描画し、炎の部分はアイソサーフェスとして描画しています。

## 基本処理
まず、フラグメントとカメラの位置を使って方向ベクトルを取得します。
そして、その方向に向かってマーチング（前進）していきます。

各ステップでボリュームのさらに奥深くへと進んでいくので、今までの色を新しいスライスの上に重ねて描画していくイメージです。
この想定でブレンドしていけば問題なく、簡単にボリュームを描画できます。

### ジッターについて
そのままではスライスとスライスの間にまったく情報がないため、木目模様のような見た目になってしまいます。
単純にステップの距離を短くしても解決しません。木目模様が細かくなって逆に目立つ場合がある上に、計算がどんどん重くなってしまいます。

そこでスマートな解決方法となるのが「ジッター」です。
カラーバンディングの解決に使われるディザリングとかなり似ています。ピクセルごとに色を混ぜるのではなく、サンプルする「深さ」を混ぜるわけです。

やり方は簡単で、レイマーチングを開始する前に、各レイの初期位置をランダムにずらすだけです。
フレームごとに違う位置にずらすことで、平均的にボリュームのすべての部分を描画できるようになります。

とはいえ、これだけだとさすがにノイズがひどくなります。
このテクニックを利用するには、必ず TAAが必要になります。

ゲーム全体で TAA を使っていない場合でも、レイマーチシェーダー内にピンポイントで TAA 的な処理を実装しないと、かなりひどい見た目になります。
（私の場合はプロジェクト全体の TAA に任せているので、シェーダー内には実装していませんが！）

## 煙部分の透明系の描画
基本的に普通のポリゴンと同じようにレイヤーを重ねて描画します。
煙の色はシェーダーのパラメータで固定し、アルファ値は煙の密度から設定します。

ボリュームの奥へ深く進みながら、ピクセルの色をどんどん加算して計算していきます。
各ステップごとに新しいレイヤーがさらに奥にある状態になるので、これまでに加算した色が新しいレイヤーの上に乗るイメージで、通常の `a`, `1-a` のアルファブレンドを行います。
このとき、乗算済みアルファで計算するのが自然ですね。

### 密度のノイズ
シミュレーションのグリッドはかなり粗いので、これだけだと最終的な見た目も粗くなってしまいます。
これもグリッドを細かくすれば解決しますが、パフォーマンスのコストが高くなるため、もっと安い方法が欲しいところです。

物理的な根拠はもちろんありませんが、ノイズを利用してさらに細かい煙に見せかけます。
単純に、密度の値からノイズの値を引いて、密度の分布を細かくするだけです。

どんなノイズを利用しても大丈夫ですが、煙系の表現には大体ボロノイノイズが使われますね。

注意点として、レイマーチシェーダー内でノイズの計算は行わないほうがいいです。
ピクセルごとに1回ノイズを計算するくらいならまあ大丈夫ですが、ここでは「ステップごと」に計算しないといけなくなるからです。
ベストな方法は、描画処理の前にタイリング可能な3Dノイズテクスチャをあらかじめ生成しておくことです。

### 煙自身の影（セルフシャドウ）
これは描画処理の前に、コンピュートシェーダーで計算しておくべきものです。
これもコンセプトは簡単で、各ボクセルからライトの方向に向かってレイマーチングをすれば、そのボクセルとライトの間に「どれくらい煙があるか」を計算できます。
それによってそのボクセルに到達する光の量が分かるので、影（セルフシャドウ）が簡単に計算できます。

### ランバートライティング
「ボリュームには表面がないのに、どうやって法線を求めてライティングすればいいの？」と思っちゃいますよね。
でも、これは意外と簡単です。

単純に密度の「勾配」を使えばいいのです（計算は描画が始まる前にコンピュートシェーダーで行います）。
これが、現在の密度におけるアイソサーフェスの法線になります。
これについては炎の描画でもっと深く使うので、後ほど詳しく説明します。

この法線を使って定番のランバート反射を計算すれば、簡単なライティングができます。

## 炎部分のアイソサーフェス描画
透明なレイヤーをどんどん重ねていく感じとは違って、しきい値となる温度を決め、その温度以上になる部分を「不透明な炎」と判断して描画します。
結果として、温度がちょうどそのしきい値になる「表面」を描画することになります。この表面をアイソサーフェス（等値面）と呼びます。

ですが、単純にレイマーチングで表面に到達しただけだと、スライスの間隔による木目模様の問題が残ります。煙と同じようにジッターで解決することもできますが、アイソサーフェスの場合はもっと良い方法があります。

透明な煙の描画と似ているように見えますが、実はまったく別の問題です。透明な場合はボリューム全体を描画したいわけですが、今回は「ある表面のみ」が欲しいのです。つまり、3次元ボリューム全体をサンプリングして重ねるのではなく、「ボリュームの中のどこに表面があるのか」を正確に知る必要があります。

ひとつ前のステップの値と位置を保持しておけば、簡単な `lerp`の計算だけで表面の位置を割り出せます。もちろん100%正確ではありませんが、描画の用途ならこれで十分です。

```hlsl
float thresh = _MinTemperature / _MaxTemperature;
if (heat > thresh)
{
    // lerp to the actual surface
    float fraction = (thresh - prevHeat) / (heat - prevHeat + 0.0001); // Runs even if heat <= thresh! This is on GPU!
    float3 surfPos = lerp(prevRayPos, rayPos, fraction);

    // ...

    break;
}
```

### マーチングキューブ法について
3次元ボリュームからアイソサーフェスを抽出する定番アルゴリズムとして「マーチングキューブ法」というものがあります。基本的にはメッシュを生成するときに使われるアルゴリズムですね。
これを使えばより正確なアイソサーフェスを素早く見つけられるかもしれませんが、今回は単純な `lerp` で十分だと思います。

理由は簡単で、どちらにしても煙を描画するためにレイマーチングを行わないといけないからです。アイソサーフェスだけが欲しいなら話は別ですが、すでにレイマーチングのループが回っているので、その「ついで」にアイソサーフェスのチェックを追加するほうが無駄がありません。

### 色の計算：黒体放射
温かい物は光を放射します。これを黒体放射と言い、炎も同じ原理です。
炎は複雑な現象ですが、基本的には炭素が高温になって光を放っています。

厳密な計算は複雑すぎるので、今回は [Tanner Hellandの近似式（概算）](https://tannerhelland.com/2012/09/18/convert-temperature-rgb-algorithm-code.html)を使用します。

ここで、1つ大きな問題に気づいたかもしれません。
わざわざアイソサーフェスを描画しているということは、表面は「すべて同じ温度」になります。すべて同じ温度ということは、すべて同じ色になってしまいます！

これも意外と簡単に解決できます。表面から「もう一歩だけ奥」に進んで、内側の温度をサンプリングすればいいのです。
表面がカメラと平行に近い場合はレイが深く潜るので、より高い温度が取得できます。逆に、表面がカメラに対してほぼ直角（輪郭に近い部分）であれば、奥に進んでも温度は表面とほぼ変わりません。

```hlsl
// Sample below the surface
float3 samplePos = saturate(surfPos + (rayDir * (_StepSize * _FireSurfaceDepth)));
float sampleHeat = SAMPLE_TEXTURE3D_LOD(_VolumeTex, sampler_VolumeTex, samplePos, 0).r;
```

## URPの影対応
煙から周りの環境に落ちる影も欲しいですよね。よりリアルになりますから。

理想を言えば、薄い煙はほとんど影を落とさず、濃い煙は固体のオブジェクトと同じくらい暗い影を落とすような、ソフトな影を描きたいところです。
もちろん計算上は可能ですが、URPのデフォルトのシャドウマップはそれに対応していません。デフォルトでは「影があるか、ないか」の0か1の情報しか扱えないのです。

本当にやりたいなら、Scriptable Render Passで自分専用のシャドウパスを作成するしかありません。可能は可能でしょうが、the juice is not worth the squeeze（労力に見合わない）ですね。

とはいえ、URPのデフォルトの影パスに煙を追加すること自体は難しくありません。
ライトの目線から描画するような処理を追加するだけです。これもまたレイマーチングを行って、密度があるしきい値を超えたら影を落とす。だけです。
