# 流体シミュレーションの基本

詳しく書くと本になりますのでざっくり書きます。

基本の仕組みだと、空間をグリッドに分けます。各セルがボクセルっと呼びます。
ボクセルごとにその流体の一部のステートの値を保存します、速度は煙の密度、温度などなど。
実際にこれは3次元テクスチャで保存しています。

流体力学の魔法をかけて、この値どう動くか分かります。
大体以下の3ステップで計算します。

1. 外部力
2. 圧力
3. 移流

1つずつみましょう

## 外部の力
ここが一番分かりやすいですね。定番の物理演算と同じように、加速の値を速度に加算するだけです。

炎と爆発のエフェクトに関して必要の力が大体2つだけです

* **重力**: 煙の密度による強く下に加速する
* **浮力**: 煙の温度による強く上に加速する

両方が簡単に同時に計算します

```hlsl
float buoyancyForce = (heat * Buoyancy) - (smokeDensity * SmokeWeight);
vel.y += buoyancyForce * DeltaTime;
```

シミュレーションに必要のはそれだけですが、見た目にいい感じにすると追加に物理の根拠がない2つの力を追加します

### カールノイズ
これはアンビエント風役になります。
現実世界だと完全なる静穏な空気が存在しません。ずっとちょっとずつ揺らしています。
その現象も再現したいのノイズを加算します。

ですが、1つだけ気を付けないと。
発散が0のノイズが必要です、さもないと圧縮します。

カールノイズがそれです。
ベクトル空間の回転の発散が必ず0ですので普通のシンプレックスノイズの回転を計算したら発散0のノイズになります。

このノイズの値が普通の力と同じく、速度に加算すればいいです

### ボルティシティ・コンファイメント（渦度閉じ込め）
これはかなり細かいところです。
シミュレーションと現実世界の流体に大きく1つの違いがあります、回転がどんどん弱くなります。
これは流体力学が不完全とかの意味ではなく、時間が不連続で計算しているのせいです。
どんなシミュレーションでも時間がステップで進みます。どこまで頑張って細かいステップにしても、非連続の事実が変わりません。

ので回転を加算する必要になります。

## 圧力の計算
非圧縮性のシミュレーションを行っていますので、圧力の役割が発散を抵抗する力です。
問題は圧力の式には解析解がありません、数値解で計算するしかないです。

そのせいですごく気を付けないとこのステップがかなり重くなります。

### ヤコビ法
一番定番な計算し方です。
現在の圧力を魔法の式に入力して、より正確な圧力が出てきます。
```
float dx = BoundsSize.x / float(Resolution.x);
float div = DivergenceWrite[id];
PressureWrite[id] = (pL + pR + pD + pU + pB + pF - (dx * dx * div)) / 6.0;
```
詳細が分からなくても使えますが、弱点が知るべきです。
隣のボクセルしか参考していないため気圧が1ボクセルごと伝播するため、グリッドの大きさと同じくらい繰り返さないといけないです。

### マルチグリッド法
隣のボクセルにしか伝播できないヤコビ法の弱点を解決する方法がマルチグリッド法です。

ボクセルが大きいほど伝播が早いがデータの制度が低くなります。
小さいボクセルがその反対、伝播が遅いけどデータの制度が高いです。
いろんなサイズのグリッドを使ったら、伝播のために粗いグリッドを使う、制度のために細かいグリッドを使う形で計算がかなり早くなります。

詳細は記載しないが、基本的にVの形に進みます。

* 一番細かいグリッドで数回ヤコビ法で圧力を伝播させる。
* ダウンサンプリングで粗いグリッドに移動して、またヤコビを使う。
* 繰り返して一番粗いグリッドまで続く
* 次はアップサンプリングして、グリッドを細かくして、またヤコビを使う。
* 一番細かいグリッドに戻ったら計算が終わり。

## 移流の計算
簡単にいうと値が速度の方向に移動させるってことです。
ボクセルがどこまで移動の計算が難しくないが、問題はどこのボクセルに移動するかです。
きれいにピッタリ隣のボクセルに移動するのはすごくあり得ません。

考え方をちょっとだけ帰ったら解決できます。
現在のボクセルがどこに行くのではなく、現在のボクセルがどこから来たって感げましょう。
どこから来たがボクセルの間にしても問題がありません、普通のトリリニアサンプリングしたら値が簡単に取得できます

### マコーマック法
これは簡単な例で説明したほうがいいと思います。
初心者の定番の物理計算はこうなるのはが多いです。

```cs
vel += acc * deltaTime;
pos += vel * deltaTime;
```

実はこれは力学敵に不正解です、問題はなんでしょう？
ブルティシティ・コンファインメントの所と同じです、時間が非連続で計算しています。

簡単な場合は正確な計算できますが基本的に完全な計算ができません。
ちなみに `acc` が変更しないケースでしたら、これが正確な計算です：

```cs
vel += acc * deltaTime;
pos += vel * deltaTime - acc * deltaTime * deltaTime / 2;
```

マコーマック法がかなり賢い方法で正確の値を近づく方法です
完全ではないが、普通の移流より大分いい感じです。
簡単にすると、以下の用に計算します

* 1ステップ前へ移流する
* そこから1ステップ後ろ移流する
* 今の所とスタート位置を比べる、その距離がエラーの2倍です（2ステップ移流したので）
* 1ステップ前の値からエラーの値を引く、正確の値と近づきます。

```hlsl
// MacCormack Advection
float3 backStep = uvwVel * DeltaTime * SimScale;
float3 backUvw = uvw - backStep;
float4 backVel = VelocityRead.SampleLevel(sampler_LinearClamp, backUvw, 0);
float3 backUvwVel = backVel.xyz / BoundsSize;

float3 forwardStep = backUvwVel * DeltaTime * SimScale;
float3 forwardUvw = backUvw + forwardStep;
float3 prevUvw = backUvw + (uvw - forwardUvw) / 2.0;

float4 advectedSmokeProps = SmokePropRead.SampleLevel(sampler_LinearClamp, prevUvw, 0);
float4 advectedVel = backVel;
```
